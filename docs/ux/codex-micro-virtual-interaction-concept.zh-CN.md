## 保留的动作语义

### Agent 状态键

- 单击：切换到对应任务，但不强制把 ChatGPT 窗口提到前台。
- 350 ms 内双击：切换任务并唤起 ChatGPT 窗口。
- 未分配的状态键：创建新任务并绑定到该槽位。
- 状态色：白色为空闲、蓝色为思考中、绿色为已完成且未读、琥珀色为等待输入、红色为错误、熄灭为未分配。
- 当前任务以对应状态色稳定显示，不做循环呼吸或闪烁，也不另造与状态色冲突的“选中颜色”。协议中的 `breath` / `shallow breath` 只保留为状态语义和悬停说明。

### 命令键

- Fast：切换 Fast 模式。
- Approve：批准当前请求。
- Decline：拒绝当前请求。
- Fork：在新任务中继续当前任务。
- Mic：按住录音、松开停止；350 ms 内双击锁定免按录音，再按一次停止。
- Send：发送输入框中的消息。
- 录音反馈：海绿色流光；语音处理中：移动白光；文本就绪：稳定白光。

### 模拟摇杆与旋钮

- 上：切换 Plan 模式。
- 右：应用历史前进。
- 下：显示或隐藏侧栏。
- 左：应用历史后退。
- 摇杆必须越过激活阈值才触发；阈值以内只做预览，回到中心取消。
- 旋钮在 Composer 控件间导航，或只调整 Reasoning；按下打开或确认当前选项。

## 虚拟版本的设计取舍

元素与布局以本地控件和 Paper 导出的 Tailwind 为准。GPT Image 2 结果不作为组件稿、布局稿或图标来源，只参考光线柔化、层次分离和动态发光的氛围：

1. 保留现有按键、旋钮、模拟摇杆和本地图标，不引入生成图中的人物头像、超大径向盘或额外控制器。
2. Agent 键沿用 Paper 的方形键帽、圆形状态光场和加号结构；状态色直接承载运行状态，避免另造冲突的选中颜色。
3. Mic 沿用 Paper 的横向宽键，只在录音、处理和就绪时叠加克制的海绿色或白色动态光。
4. 命令键严格采用 Paper 的“外层键帽、内凹圆面、玻璃高光、实体阴影”层级，并按导出比例换算到 XAML。
5. 模拟摇杆保留 Paper 的黑色圆帽和本地四向输入结构；虚拟增强只表现阈值与反馈，不更换控制元素。
6. 不使用大脑、机器人头、花朵或魔法星光代表“智能”。Reasoning 只用旋钮刻度、层级或抽象轨道表达。
7. 水晶质感以可读性为边界：发光只用于状态和反馈，轮廓与图标保持足够对比，不用大面积无意义霓虹。

## 语音键材质与状态

- 结构继续使用 Paper 的宽键帽和内凹玻璃药丸面，本地 `MIC` 矢量图标居中，按键命中区与 `ACT10` 协议不变。
- 空闲：两条交叉贝塞尔声波与点状声粒子保持低透明度，只作为玻璃纹理。
- 录音：海绿色光场轻微呼吸，声波增强，点状粒子沿曲线连续移动。
- 松开：停止循环动效并短暂闪过白色边光，提示语音正在交给 Codex 处理。
- 全部效果由 WPF 矢量和属性动画生成，不嵌入生成图片，不增加新的 UI Automation 输入路径。

## 实现约束

- 所有动作仍走现有 Micro HID／桥接协议，不因为视觉重排改回 UI Automation。
- 方向动作继续由摇杆阈值和协议事件触发，视觉方向扇区不能直接绕过输入链路。
- Paper 导出是材质、比例、元素和图标的参数基线；GPT Image 2 图片只用于光影氛围参考，不作为逐像素实现稿。
