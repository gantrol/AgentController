# Agent Controller Product Brief

> Version: 0.1
> Status: Product hypothesis — awaiting user validation
> Scope: The next two product versions

## 一句话定位

Agent Controller 是一个本地优先、控制器无关、Agent 无关的操作面：让用户用手柄、专业控制器或辅助输入设备，在不频繁切换姿势和窗口的情况下浏览 Agent 任务、查看可信状态，并安全执行语音、发送、Steer、Queue、Stop 与常用设置动作。

它不是“廉价复刻 Codex Micro”，也不是另一个完整 Agent 客户端；Micro 的六 Agent 键、六 Command 键、摇杆和旋钮构成 Codex 完整控制体验的原生设备平面，但私有协议必须被隔离在可指纹门禁、可替换的兼容层，不能渗入核心业务模型。

## 首要用户

1. **多任务 Agent 重度用户**：同时维护多个 Thread/项目，需要快速查看状态、切换任务、发送补充指令和停止错误执行；主要痛点是窗口、鼠标和上下文切换成本。
2. **语音优先、RSI 与辅助输入用户**：希望减少键鼠负担，使用按住说话、单手控制、背键或自有辅助设备完成高频动作；主要痛点是默认输入路径不舒适或不可达。
3. **演示、直播与移动姿势下的操作者**：在会议、直播、站立桌或远离键盘时控制 Agent；主要痛点是需要可记忆、可观察、低误触的物理操作和明确反馈。

这三类用户目前都是待验证假设，不以开发团队自身使用体验替代访谈证据。

## 核心任务

- 启动 Agent 客户端或将其置于前台，并确认当前控制目标。
- 跨置顶任务、置顶项目、项目与未归项目任务浏览、打开和返回。
- 查看 Thread、Turn、审批、未读与运行状态，并区分权威状态、推断状态和未知状态。
- 按住说话、编辑/清空输入、发送提示，以及在运行中 Steer、Queue 或 Stop。
- 调整 Model、Effort、Speed/Power 等当前 Agent 实际提供的设置。
- 将任意可识别物理输入映射到稳定的虚拟控制面，并安全迁移自定义 Profile。
- 在失败时看到真实执行通道、结果证据和可恢复路径，而不是只有“按键已发送”。

## 产品原则

- **控制器无关**：Xbox、8BitDo、Flydigi、Micro 类设备和辅助输入设备共享可版本化的逻辑输入层。
- **Agent 无关**：Codex 是首个生产适配器；Domain 和 Application 不包含 Codex、WPF、Win32、UIA 或私有 HID 类型。
- **权威状态优先**：Thread/Turn/审批与流式状态优先来自 App Server；UIA、rollout 和 Micro 状态保留来源、置信度与时效。
- **高风险动作可证明**：Stop、清空、批准等动作需要确认、执行证据和结果验证；不把 transport ACK 当成业务成功。
- **本地优先与默认无遥测**：不上传 prompt、任务正文或凭据；任何产品指标联网收集都必须显式 opt-in。
- **渐进迁移**：现有 WPF 客户端在新架构达到功能等价前继续维护，不用一次性重写换取表面整洁。

## 非目标

- 不替代 Codex、Claude Code 或其他 Agent 的完整桌面/IDE 客户端。
- 不做无法验证结果的通用桌面宏、反作弊规避或静默高风险自动化。
- 不在未来两个版本中建设云账户、订阅计费、团队后台或 Profile 市场。
- 不承诺兼容未经指纹验证的 Codex 私有 HID ABI，也不在稳定版自动安装实验驱动。
- 不把所有硬件能力压缩成固定 ABXY 枚举，也不让核心动作只能由背键完成。

## 未来两个版本

### 版本一：可靠性与核心契约

- 保持 v0.7 用户路径可用并执行 README 基础实机验收。
- 建立 Domain/Application/Platform Abstractions 骨架与依赖规则。
- 抽离控制器交互协调器，定义 Action、Result、Evidence 与 StateObservation。
- 完成一个 Codex App Server 垂直切片，并隔离现有 UIA/Micro 执行器。
- 不重写完整 UI、不安装虚拟 HID 驱动、不承诺 macOS 功能等价。

### 版本二：可定制控制面预览

- 提供可持久化、可迁移的自定义绑定和设备 Profile。
- 依据正式 UX 流程交付 Windows 新客户端垂直切片、菜单式动作面板和诊断体验。
- 完成原生 Micro 的设备身份 Gate、唯一 Windows 驱动候选与签名 Device Support 安装原型；未通过发行 Gate 时只进入受控测试，不要求普通用户自行编译。
- 建立 macOS shell 与平台权限/输入可行性切片，但以证据决定是否进入正式支持。
- 至少增加一个经过研究的非 Codex Agent 适配器；不以空壳插件接口宣称多 Agent 支持。

## 成功指标

- **激活**：首次运行 10 分钟内连接控制器并成功完成“打开任务、语音输入、发送或停止”中的三个核心动作。
- **可靠性**：每个动作记录 Succeeded、NotSent、AcceptedUnverified、Unsupported、Incompatible、Blocked 或 Failed，且 P0/P1 失败可定位到输入、路由、执行或验证层。
- **留存**：用户在 7 天内再次连接并完成至少一个核心动作。
- **可定制价值**：统计本地 Profile 创建与使用情况；只有用户显式 opt-in 才汇总匿名指标。
- **可恢复性**：失败路径能给出下一步，诊断导出不包含 prompt、任务正文或凭据。

## 需要验证的关键假设

- 控制器带来的任务切换和语音操作收益足以形成每周重复使用，而不只是新鲜感。
- 至少一个首要用户群愿意为签名发行版、稳定更新、支持或设备适配付费。
- 用户能理解菜单式分层导航、模式切换和高风险确认，不会因隐藏状态增加误触。
- App Server 足以持有 Thread/Turn 等语义主链；目标用户是否愿意为 Full Micro mode 安装一次已签名 Device Support，或改用免驱硬件桥，是必须验证的采用门槛。无法安装时只能提供明确标记的 Limited mode，不能用 UIA 宣称等价。

验证方式为至少 10 次目标用户访谈、基础任务可用性测试、失败日志和可证伪结论；Star、下载量或开发者自用不能单独通过这些假设。

## 尚未决定

- PolyForm Noncommercial、标准开源许可证或双重许可的最终选择。
- 个人签名发行版、团队支持和 OEM/Profile 合作的具体付费组合。
- macOS 最低版本、虚拟 HID 发行方式和是否值得承担签名/权限支持成本。
- 第二个生产 Agent 适配器及其官方状态/动作接口质量。
